# Zetta Lab 2025
## DESAFIO I: Desenvolvimento de Software

### 1. Descrição do Projeto (Backend)
Você deverá desenvolver uma API de um sistema simples de gerenciamento de tarefas (to-do list), com autenticação simples, persistência em banco de dados e documentação básica.

### 2. Funcionalidades
O sistema deve permitir que usuários possam:
* Se cadastrar e se autenticar;
* Criar, listar, atualizar e excluir tarefas;
* Listar tarefas filtrando por status (pendente, concluída);
* Armazenar tudo em um banco de dados.

### 3. Entidades e Campos Mínimos
* **Tarefa:** nome, descrição e status.
* **Usuário:** nome, email e senha.

> *Seja criativo!*

---

### 4. Requisitos Técnicos

#### Obrigatórios
* **Linguagem backend:** à sua escolha (ex: Python, Java, JavaScript/Node.js, Go, etc.).
* **Framework web:** à sua escolha (ex: Flask, Spring, Express, etc.).
* **Persistência:** banco de dados relacional (SQLite, PostgreSQL, MySQL, etc.).
* **Arquitetura:** API com rotas bem definidas, seguindo o padrão REST.

#### Desejáveis
* Autenticação por token (ex: JWT simples);
* Organização de código por camadas (controllers, services, models);
* Testes unitários e/ou de integração;
* Uso de Docker para subir a aplicação.

---

### 5. Entregáveis
* Documentação da API (ex: README + exemplos de uso via cURL ou Postman, Swagger, etc.);
* Código-fonte versionado no Git;
* Modelo de Entidade e Relacionamento;
* Script com a criação do banco de dados.

**Importante:** As boas práticas recomendadas no e-book e nas mentorias serão observadas durante a avaliação.
